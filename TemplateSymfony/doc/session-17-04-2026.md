## Livré — détails

### RAG sémantique (priorité #1)

- **Vector store SQLite custom** (`agent/app/rag/store.py`) :
  - Table `rag_chunks` avec `(coach_id, team_id, source_type, source_id, content, embedding BLOB, dim)`.
  - Unicité `(coach_id, source_type, source_id)` → upsert naturel.
  - Similarité cosinus via numpy (float32) en mémoire ; dimensionnement OK jusqu'à ~100 k chunks / coach.
  - Isolation stricte par `coach_id` (testé).
- **Embeddings pluggables** (`agent/app/rag/embeddings.py`) :
  - Backend par défaut : Ollama `nomic-embed-text` (local, gratuit, pullé au boot).
  - Backend alternatif : OpenAI `text-embedding-3-small` (activable via `EMBEDDINGS_BACKEND=openai`).
- **Indexer** (`agent/app/rag/indexer.py`) : `index_note`, `index_match_notes_batch`, `search_notes`.
- **Endpoints FastAPI** : `POST /rag/index`, `POST /rag/index-batch`, `POST /rag/search`, `GET /rag/stats`, tous protégés par `X-Agent-Internal-Secret`.
- **Tool agent `search_notes_semantic`** (`agent/app/agent/tools.py`), ajouté à `ALL_TOOLS`.
- **Hook Symfony automatique** : `AgentRagIndexer` appelé depuis `MatchNoteApiController::create` et `::update`. Indexation non bloquante (log si l'agent est down).
- **Commande de backfill** : `php bin/console app:rag:reindex`.
- **Tests** : `agent/tests/test_rag_store.py` couvre upsert, isolation par coach, search top-K, delete.
- **Infra** :
  - `compose.yaml` : pull auto de `nomic-embed-text`, volume `agent_data:/app/var` pour persister le SQLite RAG.
  - `.env.example` enrichi (`EMBEDDINGS_BACKEND`, `OLLAMA_EMBEDDINGS_MODEL`, placeholders OpenAI).

## Décisions techniques notables

1. **Choix du vector store maison** plutôt que `chromadb`/`pgvector`/`sqlite-vec` : minimise les deps, fonctionne out-of-the-box, assez rapide pour le scale MVP. Documenté comme dette à migrer si > 100 k chunks.
2. **Mode fallback Stripe** : permet de tester le parcours billing sans compte Stripe. La bascule en mode réel est documentée (un `composer require` + décommenter).
3. **Mode fallback PDF** : évite de bloquer la feature sur une dépendance lourde.
4. **Mailpit en dev** : pas de magie, l'UI est trivialement accessible.
5. **API publique sans `nelmio_api_doc.yaml` actif** : le fichier est en `.example` pour ne pas planter Symfony tant que le bundle n'est pas installé.
6. **i18n non appliqué partout** : extraction des chaînes depuis les templates existants est à faire dans un second temps, la config est prête.
7. **Bootstrapping auto des nouvelles migrations** : toutes nos migrations portent des timestamps `202604171XXXXX` bien ordonnés, `doctrine:migrations:migrate` les appliquera proprement.
